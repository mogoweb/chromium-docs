Pages and Endpoints

Main public pages



 PathPurpose Query parameters
 / View recent bugs and top alerts days: Number of days to show anomalies and bugs for (optional).

 sheriff: Sheriff to show anomalies for (optional)

 num_changes: The number of improvements/regressions to list.

 /alerts View current outstanding alerts sheriff: A sheriff rotation name, defaults to Chromium Perf Sheriff.
 triaged: Whether to include recent already-triaged alerts.
 improvements: Whether to include improvement alerts.
 sortby: A field in the alerts table to sort rows by.
 sortdirection: Direction to sort, either "up" or "down".
 /bisect_stats View bisect job success rate stats. None.
 /debug_alert Experiment with the alerting function, or diagnose why and when an alert would occur at some place.
 test_path: Full test path (Master/bot/benchmark/...) to get points for.
 rev: A revision to center the graph on.
 num_before: Number of points to fetch before rev.
 num_after: Number of points to fetch starting from rev.
 config: JSON containing custom thresholds parameters.
 /embed Embed-able version of /report masters: Comma-separated list of master names
 bots: Comma-separated list of bot names.
 tests: Comma-separated list of test paths starting from benchmark name.
 rev: Revision number (optional).

 num_points: Number of points to plot (optional).

 start_rev: Starting evision number (optional).

 end_rev: Ending revision number (optional).

 /get_logs Show the content of a "quick-log" from the datastore. log_namespace: Namespace of log to retrieve.
 log_name: Name of log to retrieve.
 log_filter: Regex string to filter logs (optional).
 log_size: Maximum number of logs to get.
 after_index: Get the logs after this index.
 before_index: Get the logs before this index.
 /graph_csv Download data for one series at a time  num_points: The number of points to download
 attr: Comma-separated list of attributes to download
 /group_report  View graphs for a set of alerts bug_id: Bug ID to view alerts for.
 rev: Chromium commit position to view alerts for.
 keys: Comma-separated list URL-safe keys, each represents one alert
 /new_points View recently-added points for some set of tests, and verify whether or not data was received. num_points: Max number of points to fetch.
 pattern: A test path pattern (Master/bot/benchmark/...) with wildcards to match.
 max_tests: Maximum number of tests that match the pattern to fetch.
 /report Browse graphs and compare charts across platforms.  masters: Comma-separated list of master names
 bots: Comma-separated list of bot names.
 tests: Comma-separated list of test paths starting from benchmark name.
 rev: Revision number (optional).

 num_points: Number of points to plot (optional).

 start_rev: Starting evision number (optional).

 end_rev: Ending revision number (optional).

 checked: Series to check. Could be "core" (important + ref) or "all".

 /run_element_tests Run some tests for custom elements (limited). element: Custom polymer element name.
 /set_warning_message Sets a warning message for users of the /report and /group_report pages about outages and planned maintenance. None.
 /size_viewer View executable size information and breakdown. path: Path to file
 executable: Executable name to show data for.
 revision: Revision to get data for, may be a commit hash.
 /stats View and generate stats about alert volume.  key: URL-safe key of existing previously generated stats group.
